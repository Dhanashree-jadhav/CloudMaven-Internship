## ☁️ What is Cloud Computing?

Cloud computing is the **on-demand delivery of computing services such as servers, storage, databases, networking, and software over the internet**, with flexible pricing based on usage.

👉 In simple terms:
Organizations no longer need to own physical infrastructure—they can **use resources whenever required and release them when not needed**.

👉 **Example:**
A developer can launch a web server in minutes instead of purchasing hardware and waiting days for setup.

---

## ☁️ Why Cloud is Important?

Cloud technology addresses several real-world challenges:

* Eliminates high upfront infrastructure costs
* Enables fast resource provisioning
* Simplifies scaling of applications
* Reduces operational and maintenance effort
* Improves system availability and reliability

👉 **Real-world Scenario:**
A startup launching a new app deploys on cloud and pays only for usage. As users grow, infrastructure scales automatically without downtime.

---

## ☁️ Traditional Infrastructure vs Cloud

👉 **Example:**
An e-commerce platform during a festive sale can instantly scale servers in cloud, whereas traditional systems may crash due to fixed capacity.

| Factor      | Traditional IT          | Cloud               |
| ----------- | ----------------------- | ------------------- |
| Cost        | High initial investment | Pay-as-you-go       |
| Setup Time  | Weeks or months         | Minutes             |
| Scaling     | Complex                 | Simple and flexible |
| Maintenance | Manual effort           | Managed by provider |

## ☁️ Cloud Deployment Models

👉 **Real-world Example:**
A banking system stores sensitive data in a private cloud while hosting its mobile app on public cloud—this is a hybrid model.

### Public Cloud

* Infrastructure owned and managed by cloud providers
* Accessible over the internet

### Private Cloud

* Dedicated infrastructure for a single organization
* Offers more control and customization

### Hybrid Cloud

* Combines public and private cloud environments
* Balances security and scalability

---

## ☁️ Cloud Service Models

👉 **Real-world Scenario:**

* A startup quickly deploying an app → uses **PaaS**
* A company needing full control → uses **IaaS**
* End users accessing Gmail → using **SaaS**

### IaaS (Infrastructure as a Service)

* Virtual machines, storage, networking
* User manages OS and applications

### PaaS (Platform as a Service)

* Platform to build and deploy applications
* No infrastructure management required

### SaaS (Software as a Service)

* Ready-to-use software via internet
* Example: Gmail, Google Docs

## ☁️ Core Cloud Concepts

👉 **Real-world Example:**
During a live streaming app (e.g., sports match):

* **Scalability** handles sudden traffic spikes

* **Elasticity** adds/removes servers automatically

* **High Availability** ensures service is always accessible

* **Fault Tolerance** keeps system running even if a server fails

* **Scalability**: Handles increasing workload

* **Elasticity**: Auto scaling based on demand

* **High Availability**: Minimal downtime

* **Fault Tolerance**: Continues working despite failures

---

## ☁️ Shared Responsibility Model

👉 **Real-world Scenario:**
If a company stores user data in the cloud:

* Provider secures infrastructure
* Company must secure data, users, and application settings

### Cloud Provider Responsibilities:

* Data centers
* Physical hardware
* Networking

### Customer Responsibilities:

* Data security
* Access control
* Application configuration

## ☁️ AWS Global Infrastructure

👉 **Real-world Scenario:**
If an application runs in multiple Availability Zones and one data center fails, users still experience no downtime.

* **Regions**: Geographical locations (e.g., Mumbai, US, Europe)
* **Availability Zones (AZs)**: Multiple isolated data centers in a region

👉 Benefits:

* High availability
* Disaster recovery
* Better performance
